## 1. 📝 题目描述

- [leetcode](https://leetcode.cn/problems/minimum-recolors-to-get-k-consecutive-black-blocks/)

给你一个长度为 `n` 下标从 0 开始的字符串 `blocks` ，`blocks[i]` 要么是 `'W'` 要么是 `'B'`，表示第 `i` 块的颜色。字符 `'W'` 和 `'B'` 分别表示白色和黑色。

给你一个整数 `k`，表示想要连续黑色块的数目。

每一次操作中，你可以选择一个白色块将它涂成黑色块。

请你返回至少出现一次连续 `k` 个黑色块的最少操作次数。

---

示例 1：

```txt
输入：blocks = "WBBWWBBWBW", k = 7
输出：3

解释：
一种得到 7 个连续黑色块的方法是把第 0 ，3 和 4 个块涂成黑色。
得到 blocks = "BBBBBBBWBW"。
可以证明无法用少于 3 次操作得到 7 个连续的黑块。
所以我们返回 3。
```

---

示例 2：

```txt
输入：blocks = "WBWBBBW", k = 2
输出：0

解释：
不需要任何操作，因为已经有 2 个连续的黑块。
所以我们返回 0。
```

---

提示：

- `n == blocks.length`
- `1 <= n <= 100`
- `blocks[i]` 要么是 `'W'` ，要么是 `'B'`。
- `1 <= k <= n`

- 时间复杂度：$O(N)$，其中 N 是字符串 blocks 的长度
- 空间复杂度：$O(1)$，只使用常数额外空间

算法思路：

- 维护一个长度为 k 的滑动窗口，统计窗口中白色块的数量
- 初始化第一个窗口的白色块计数
- 窗口向右滑动时，新加入的块如果是白色则计数加 1，移出的块如果是白色则计数减 1
- 记录所有窗口中白色块数量的最小值，即为最少涂色次数

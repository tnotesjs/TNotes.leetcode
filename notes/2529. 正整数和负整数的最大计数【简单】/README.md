## 1. 📝 题目描述

- [leetcode](https://leetcode.cn/problems/maximum-count-of-positive-integer-and-negative-integer/)

给你一个按 非递减顺序 排列的数组 `nums` ，返回正整数数目和负整数数目中的最大值。

- 换句话讲，如果 `nums` 中正整数的数目是 `pos` ，而负整数的数目是 `neg` ，返回 `pos` 和 `neg`二者中的最大值。

---

注意： `0` 既不是正整数也不是负整数。

---

示例 1：

```txt
输入：nums = [-2,-1,-1,1,2,3]
输出：3

解释：
共有 3 个正整数和 3 个负整数。计数得到的最大值是 3。
```

---

示例 2：

```txt
输入：nums = [-3,-2,-1,0,0,1,2]
输出：3

解释：
共有 2 个正整数和 3 个负整数。计数得到的最大值是 3。
```

---

示例 3：

```txt
输入：nums = [5,20,66,1314]
输出：4

解释：
共有 4 个正整数和 0 个负整数。计数得到的最大值是 4。
```

---

提示：

- `1 <= nums.length <= 2000`
- `-2000 <= nums[i] <= 2000`
- `nums` 按 非递减顺序 排列。

进阶： 你可以设计并实现时间复杂度为 `O(log(n))` 的算法解决此问题吗？

- 时间复杂度：$O(\log N)$，其中 N 是数组长度，使用两次二分查找
- 空间复杂度：$O(1)$，只使用常数额外空间

算法思路：

- 利用数组已排序的特性，使用二分查找定位关键位置
- 查找第一个大于 0 的位置 `firstPositive`，则正整数数量为 `n - firstPositive`
- 查找第一个大于等于 0 的位置 `firstNonNegative`，则负整数数量为 `firstNonNegative`
- 返回正整数和负整数数量的最大值

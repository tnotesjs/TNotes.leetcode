## 1. 📝 题目描述

- [leetcode](https://leetcode.cn/problems/balanced-binary-tree/)

给定一个二叉树，判断它是否是平衡二叉树。

> 平衡二叉树：是指该树所有节点的左右子树的高度相差不超过 1。

---

示例 1：

![img](https://cdn.jsdelivr.net/gh/tnotesjs/imgs@main/2025-09-09-13-22-49.png)

```txt
输入：root = [3, 9, 20, null, null, 15, 7]
输出：true
```

---

示例 2：

![img](https://cdn.jsdelivr.net/gh/tnotesjs/imgs@main/2025-09-09-13-23-06.png)

```txt
输入：root = [1, 2, 2, 3, 3, null, null, 4, 4]
输出：false
```

---

示例 3：

```txt
输入：root = []
输出：true
```

---

提示：

- 树中的节点数在范围 `[0, 5000]` 内
- `-10^4 <= Node.val <= 10^4`

- 时间复杂度：$O(n^2)$，自顶向下时对每个节点都要重复计算子树高度，最坏情况下会重复遍历大量节点
- 空间复杂度：$O(n)$，递归调用栈最深可达树的高度，链状树下为 $O(n)$

算法思路：

- 先计算当前节点左子树和右子树的高度
- 若高度差超过 1，说明当前节点不平衡，直接返回 `false`
- 同时递归检查左子树和右子树是否都平衡，三者同时满足才返回 `true`
- 缺点是同一棵子树的高度会被重复计算，整体效率低于自底向上写法

## 3. 🎯 s.2 - 自底向上

::: code-group

<<< ./solutions/2/1.js [js]

<<< ./solutions/2/1.py [py]

:::

- 时间复杂度：$O(n)$，每个节点只访问一次
- 空间复杂度：$O(n)$，递归调用栈最深可达树的高度，链状树下为 $O(n)$

算法思路：

- 自底向上计算每个节点的高度，同时检查平衡性
- 若某棵子树已经不平衡，直接返回 `-1` 作为哨兵值，逐层向上传递，避免重复判断
- 若左右子树高度差超过 1，同样返回 `-1`
- 否则返回当前节点的真实高度 `max(left, right) + 1`
- 最终判断整棵树是否平衡，只需看根节点返回值是否为 `-1`

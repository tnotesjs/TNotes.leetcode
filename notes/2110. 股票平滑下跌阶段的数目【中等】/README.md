## 1. 📝 题目描述

- [leetcode](https://leetcode.cn/problems/number-of-smooth-descent-periods-of-a-stock/)

给你一个整数数组 `prices`，表示一支股票的历史每日股价，其中 `prices[i]` 是这支股票第 `i` 天的价格。

一个 平滑下降的阶段 定义为：对于 连续一天或者多天，每日股价都比 前一日股价恰好少 `1`，这个阶段第一天的股价没有限制。

请你返回 平滑下降阶段 的数目。

---

示例 1：

```txt
输入：prices = [3,2,1,4]
输出：7
解释：总共有 7 个平滑下降阶段：
[3], [2], [1], [4], [3,2], [2,1] 和 [3,2,1]
注意，仅一天按照定义也是平滑下降阶段。
```

示例 2：

```txt
输入：prices = [8,6,7,7]
输出：4
解释：总共有 4 个连续平滑下降阶段：[8], [6], [7] 和 [7]
由于 8 - 6 ≠ 1，所以 [8,6] 不是平滑下降阶段。
```

示例 3：

```txt
输入：prices = [1]
输出：1
解释：总共有 1 个平滑下降阶段：[1]
```

---

提示：

- `1 <= prices.length <= 10^5`
- `1 <= prices[i] <= 10^5`

- 时间复杂度：$O(n)$，其中 n 是 prices 数组的长度
- 空间复杂度：$O(1)$，只使用了常数级别的额外空间

算法思路：

- 平滑下跌阶段要求连续元素之差恰好为 1，维护当前连续下跌长度 count
- 如果当前价格比前一天恰好少 1，则 count++，否则重置为 1
- 以每个位置结尾的平滑下跌子数组数量恰好等于 count，累加到结果中

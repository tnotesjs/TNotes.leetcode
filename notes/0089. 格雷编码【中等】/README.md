## 1. 题目描述

- [leetcode](https://leetcode.cn/problems/gray-code/)

n 位格雷码序列是一个由 `2^n` 个整数组成的序列，其中：

- 每个整数都在范围 `[0, 2^n - 1]` 内（含 `0` 和 `2^n - 1`）
- 第一个整数是 `0`
- 一个整数在序列中出现不超过一次
- 每对相邻整数的二进制表示恰好一位不同，且
- 第一个和最后一个整数的二进制表示恰好一位不同

给你一个整数 `n`，返回任一有效的 n 位格雷码序列。

---

示例 1：

```txt
输入：n = 2
输出：[0, 1, 3, 2]
```

解释：

- [0, 1, 3, 2] 的二进制表示是 [00, 01, 11, 10]。
  - 00 和 01 有一位不同
  - 01 和 11 有一位不同
  - 11 和 10 有一位不同
  - 10 和 00 有一位不同
- [0, 2, 3, 1] 也是一个有效的格雷码序列，其二进制表示是 [00, 10, 11, 01]。
  - 00 和 10 有一位不同
  - 10 和 11 有一位不同
  - 11 和 01 有一位不同
  - 01 和 00 有一位不同

---

示例 2：

```txt
输入：n = 1
输出：[0, 1]
```

---

提示：

- `1 <= n <= 16`

- 时间复杂度：$O(2^n)$，生成 $2^n$ 个格雷编码
- 空间复杂度：$O(1)$，不计输出数组，只使用常数额外空间

算法思路：

- 格雷编码公式：$G(i) = i \oplus (i >> 1)$，即将整数 `i` 与其右移一位的结果异或
- 该公式保证相邻两个编码恰好只有一位不同
- 遍历 $[0, 2^n)$，对每个 `i` 应用公式即可

## 3. 证明公式成立【AI】

设公式生成的第 `i` 个数为：

```js
G(i) = i ^ (i >> 1)
```

也就是：

```txt
格雷码 = 原数 ^ 原数右移一位
```

下面证明它生成的是合法的 n 位格雷码序列。

### 3.1. 每个数都在 `[0, 2^n - 1]` 内

当：

```txt
0 <= i < 2^n
```

时，`i` 是一个 n 位二进制数。

`i >> 1` 也是一个不超过 n 位的二进制数。

两个 n 位数异或，结果仍然不会超过 n 位，所以：

```txt
0 <= G(i) < 2^n
```

因此所有数都在合法范围内。

### 3.2. 第一个整数是 0

当 `i = 0` 时：

```txt
G(0) = 0 ^ (0 >> 1)
     = 0 ^ 0
     = 0
```

所以序列第一个数一定是 `0`。

### 3.3. 不会出现重复数字

设 `i` 的二进制是：

```txt
b[n-1] b[n-2] ... b[1] b[0]
```

那么：

```txt
i >> 1 = 0 b[n-1] b[n-2] ... b[1]
```

因此：

```txt
G(i) = i ^ (i >> 1)
```

它的每一位为：

```txt
g[n-1] = b[n-1]
g[k]   = b[k+1] ^ b[k]    其中 0 <= k < n - 1
```

也就是说，格雷码的最高位等于原数最高位，其余每一位是原数相邻两位的异或。

现在反过来，如果已知格雷码 `g`，可以恢复原数 `b`：

```txt
b[n-1] = g[n-1]
b[k] = b[k+1] ^ g[k]
```

也就是说，从最高位开始，可以唯一地恢复出原来的 `i`。

因此不同的 `i` 不可能得到相同的 `G(i)`。

所以序列中没有重复数字。

### 3.4. 相邻两个数只差一位

这是核心。

考虑相邻两个整数：

```txt
i 和 i + 1
```

假设 `i` 的二进制末尾有 `c` 个连续的 `1`。

那么 `i` 的低位形态一定是：

```txt
... 0 1 1 1 ... 1
      ↑  后面有 c 个 1
```

加一以后变成：

```txt
... 1 0 0 0 ... 0
```

也就是说：

- 第 `c` 位从 `0` 变成 `1`
- 第 `0` 到第 `c - 1` 位从 `1` 变成 `0`
- 更高位不变

现在看格雷码的第 `k` 位：

```txt
g[k] = b[k+1] ^ b[k]
```

加一之后：

### 3.5. 更高位 `k > c`

对应的原二进制位没有变化，所以格雷码位不变。

### 3.6. 更低位 `k < c`

这些位虽然发生了变化，但相邻两位是一起变化的：

```txt
1 ^ 1 变成 0 ^ 0
```

或者：

```txt
0 ^ 1 变成 1 ^ 0
```

结果都不变。

### 3.7. 只有第 `c` 位

第 `c` 位的格雷码是：

```txt
g[c] = b[c+1] ^ b[c]
```

其中 `b[c]` 从 `0` 变成 `1`，而 `b[c+1]` 没变。

所以：

```txt
g[c]
```

一定会翻转。

因此：

```txt
G(i) 和 G(i + 1)
```

恰好只有第 `c` 位不同。

所以相邻两个格雷码只差一位。

### 3.8. 第一个和最后一个也只差一位

第一个是：

```txt
G(0) = 0
```

最后一个对应：

```txt
i = 2^n - 1
```

二进制是：

```txt
111...111
```

右移一位是：

```txt
011...111
```

异或得到：

```txt
100...000
```

也就是：

```txt
G(2^n - 1) = 2^(n - 1)
```

它和 `0` 的二进制相比，只有最高位不同：

```txt
000...000
100...000
```

所以首尾也只差一位。

---

因此公式：

```js
i ^ (i >> 1)
```

生成的序列满足格雷码的全部条件，所以代码成立：

```js
var grayCode = function (n) {
  const result = []
  for (let i = 0; i < 1 << n; i++) {
    result.push(i ^ (i >> 1))
  }
  return result
}
```

## 4. 格雷编码的应用场景都有哪些？【AI】

格雷编码的核心价值是：相邻数值只改变 1 个二进制位。因此它特别适合“状态连续变化、但多位同时翻转可能造成误判”的场景。

| 应用场景                | 为什么使用格雷编码                                                                                                                                     |
| ----------------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------ |
| 旋转编码器 / 位置传感器 | 电机、机械旋钮、机器人关节转动时，普通二进制可能多位同时变化；传感器处在临界位置时容易读到错误组合。格雷码相邻位置仅变一位，误差通常最多落到相邻位置。 |
| 绝对值编码器            | 工业伺服电机、CNC、机械臂中，编码盘常直接采用格雷码刻度，断电后仍可读取当前位置。                                                                      |
| 模数转换器（ADC）       | Flash ADC、比较器阵列等电路中，多个比较器切换不同步会产生瞬态错误。将温度码或二进制结果转换为格雷码，可降低切换毛刺影响。                              |
| 异步 FIFO               | 数字电路中，写时钟和读时钟不同时，读写指针跨时钟域同步容易出现亚稳态。使用格雷码指针后，每次地址递增只改变一位，更容易安全同步。                       |
| 数字通信调制            | QPSK、QAM、PSK 等星座图常采用 Gray mapping。相邻星座点只差一位，因此一次误判到邻近符号时，通常只造成 1 bit 错误。                                      |
| 卡诺图（K-map）         | 卡诺图行列编号使用格雷码，例如 `00, 01, 11, 10`，使相邻单元只差一个变量，便于合并最小项并化简逻辑表达式。                                              |
| 状态机设计              | 某些有限状态机采用格雷码状态编码，减少状态切换时的翻转位数，从而降低毛刺、功耗或竞争冒险。                                                             |
| 遗传算法与优化          | 把连续变量映射为二进制染色体时，普通二进制会出现“汉明悬崖”：数值相邻但编码差很多位。格雷码能让相邻数值的编码更连续，改善局部搜索。                     |
| 图像与结构光三维测量    | 投射格雷码条纹并由相机识别，可为像素建立唯一位置编号，用于深度恢复、工业检测和三维扫描。                                                               |
| 数据存储与计数器        | 某些高速计数器、低功耗电路中，格雷码可减少总线翻转数量，降低瞬态干扰与动态功耗。                                                                       |

例如普通二进制从 `3` 增加到 `4`：

```txt
3 = 011
4 = 100
```

三位同时变化，硬件如果没有完全同步，可能短暂读到错误值。

而格雷码序列中对应的相邻状态可能是：

```txt
010 -> 110
```

只变化一位，因此更稳定。

需要注意：格雷码不是纠错码，它不能自动修复错误；它主要减少“相邻状态切换时，多位不同步变化”带来的歧义。

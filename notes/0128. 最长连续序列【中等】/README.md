## 1. 📝 题目描述

- [leetcode](https://leetcode.cn/problems/longest-consecutive-sequence/)

给定一个未排序的整数数组 `nums`，找出数字连续的最长序列（不要求序列元素在原数组中连续）的长度。

请你设计并实现时间复杂度为 `O(n)` 的算法解决此问题。

---

示例 1：

```txt
输入：nums = [100,4,200,1,3,2]
输出：4
解释：最长数字连续序列是 [1, 2, 3, 4]。它的长度为 4。
```

示例 2：

```txt
输入：nums = [0,3,7,2,5,8,4,6,0,1]
输出：9
```

示例 3：

```txt
输入：nums = [1,0,1,2]
输出：3
```

---

提示：

- `0 <= nums.length <= 10^5`
- `-10^9 <= nums[i] <= 10^9`

- 时间复杂度：$O(n)$，其中 $n$ 是数组长度（C 版本为 $O(n \log n)$ 因为使用排序代替哈希集合）
- 空间复杂度：$O(n)$，哈希集合存储所有元素（C 版本为 $O(1)$ 原地排序）

算法思路：

- 将所有元素放入哈希集合中，实现 $O(1)$ 查找
- 遍历集合中的每个元素 `num`，只有当 `num - 1` 不在集合中时，才从 `num` 开始向后计数连续序列长度
- 这样每个元素最多被访问两次，保证了总体 $O(n)$ 的时间复杂度

## 1. 📝 题目描述

- [leetcode](https://leetcode.cn/problems/longest-consecutive-sequence/)

给定一个未排序的整数数组 `nums`，找出数字连续的最长序列（不要求序列元素在原数组中连续）的长度。

请你设计并实现时间复杂度为 `O(n)` 的算法解决此问题。

---

示例 1：

```txt
输入：nums = [100, 4, 200, 1, 3, 2]
输出：4
```

解释：最长数字连续序列是 [1, 2, 3, 4]。它的长度为 4。

---

示例 2：

```txt
输入：nums = [0, 3, 7, 2, 5, 8, 4, 6, 0, 1]
输出：9
```

---

示例 3：

```txt
输入：nums = [1, 0, 1, 2]
输出：3
```

---

提示：

- `0 <= nums.length <= 10^5`
- `-10^9 <= nums[i] <= 10^9`

- 时间复杂度：$O(n)$，其中 $n$ 是数组长度
- 空间复杂度：$O(n)$，哈希集合存储所有元素

算法思路：

- 将所有元素放入哈希集合中，实现 $O(1)$ 查找
- 遍历集合中的每个元素 `num`，只有当 `num - 1` 不在集合中时，才从 `num` 开始向后计数连续序列长度
- 这样每个元素最多被访问两次，保证了总体 $O(n)$ 的时间复杂度

### 2.1. 为什么时间复杂度是 `O(n)`？【AI】

关键在于 **每个元素最多被访问两次**，虽然看起来有嵌套循环。

假设数组是 `[100, 4, 200, 1, 3, 2]`，放入 Set 后遍历：

- `num = 100`：`99` 不在 Set -> 从 100 开始数：101 不在 -> 序列长度 1
- `num = 4`：`3` 在 Set -> **跳过**
- `num = 200`：`199` 不在 -> 从 200 开始数：201 不在 -> 长度 1
- `num = 1`：`0` 不在 -> 从 1 开始数：2->3->4 都在 -> 序列长度 4
- `num = 3`：`2` 在 Set -> **跳过**
- `num = 2`：`1` 在 Set -> **跳过**

注意 `4`, `3`, `2` 都被跳过了，因为只有**序列的起点**（`num - 1` 不存在）才会触发内层计数。同一个序列里的元素不会各自重新数一遍。

总结：

| 条件               | 操作                                      |
| ------------------ | ----------------------------------------- |
| `num - 1` 在 Set   | 跳过，什么也不做                          |
| `num - 1` 不在 Set | 从 `num` 开始往上数，至多把整个序列走一遍 |

每个元素最多被「跳过检查」访问一次，再被「所属序列的计数过程」访问一次，总体 $O(n + n) = O(n)$。

## 1. 📝 题目描述

- [leetcode](https://leetcode.cn/problems/minimum-right-shifts-to-sort-the-array/)

给你一个长度为 `n` 下标从 0 开始的数组 `nums`，数组中的元素为互不相同的正整数。请你返回让 `nums` 成为递增数组的最少右移次数，如果无法得到递增数组，返回 `-1`。

一次右移指的是同时对所有下标进行操作，将下标为 `i` 的元素移动到下标 `(i + 1) % n` 处。

---

示例 1：

```txt
输入：nums = [3,4,5,1,2]
输出：2

解释：
第一次右移后，nums = [2,3,4,5,1]。
第二次右移后，nums = [1,2,3,4,5]。
现在 nums 是递增数组了，所以答案为 2。
```

---

示例 2：

```txt
输入：nums = [1,3,5]
输出：0

解释：
nums 已经是递增数组了，所以答案为 0。
```

---

示例 3：

```txt
输入：nums = [2,1,4]
输出：-1

解释：
无法将数组变为递增数组。
```

---

提示：

- `1 <= nums.length <= 100`
- `1 <= nums[i] <= 100`
- `nums` 中的整数互不相同。

- 时间复杂度：$O(n)$，n 是数组 nums 的长度
- 空间复杂度：$O(1)$，只使用常数额外空间

算法思路：

- 遍历数组查找断点位置（即 nums[i] > nums[i+1] 的位置）
- 如果没有断点，说明数组已经递增，返回 0
- 如果有多个断点，无法通过右移使数组递增，返回 -1
- 如果只有一个断点，检查 nums[n-1] 是否小于等于 nums[0]，不满足则返回 -1
- 满足条件时，右移次数 = n - (断点位置 + 1)

## 1. 题目描述

- [leetcode](https://leetcode.cn/problems/find-the-duplicate-number/)

给定一个包含 `n + 1` 个整数的数组 `nums`，其数字都在 `[1, n]` 范围内（包括 `1` 和 `n`），可知至少存在一个重复的整数。

假设 `nums` 只有一个重复的整数，返回这个重复的数。

你设计的解决方案必须不修改数组 `nums` 且只用常量级 `O(1)` 的额外空间。

---

示例 1：

```txt
输入：nums = [1, 3, 4, 2, 2]
输出：2
```

---

示例 2：

```txt
输入：nums = [3, 1, 3, 4, 2]
输出：3
```

---

示例 3 :

```txt
输入：nums = [3, 3, 3, 3, 3]
输出：3
```

---

提示：

- `1 <= n <= 10^5`
- `nums.length == n + 1`
- `1 <= nums[i] <= n`
- `nums` 中只有一个整数出现两次或多次，其余整数均只出现一次

---

进阶：

- 如何证明 `nums` 中至少存在一个重复的数字?
- 你可以设计一个线性级时间复杂度 `O(n)` 的解决方案吗？

- 时间复杂度：$O(n)$，其中 $n$ 是数组长度
- 空间复杂度：$O(1)$

算法思路：

- 将数组视为链表（索引 $\to$ 值的映射），重复数即环的入口
- Floyd 快慢指针：先找到环内相遇点，再从起点和相遇点同时出发找环入口

## 3. s.1 - 题解解释【AI】

这道题的核心是：**把数组看成一张隐式链表，然后使用 Floyd 快慢指针寻找环的入口。**

### 3.1. 为什么一定存在重复数字？

数组中一共有 `n + 1` 个数字，但每个数字只能取 `[1, n]` 中的值，一共只有 `n` 种可能。

根据**鸽巢原理**：

> 把 `n + 1` 个元素放入 `n` 个位置，至少有一个位置会放入两个或更多元素。

因此数组中一定存在重复数字。

---

### 3.2. 如何把数组看成链表？

把数组下标视为链表节点，并定义：

```js
下一个节点 = nums[当前下标]
```

也就是：

```js
i -> nums[i]
```

例如：

```txt
nums = [1, 3, 4, 2, 2]
```

对应关系是：

```txt
下标 0 -> nums[0] = 1
下标 1 -> nums[1] = 3
下标 3 -> nums[3] = 2
下标 2 -> nums[2] = 4
下标 4 -> nums[4] = 2
```

从下标 `0` 开始移动：

```txt
0 -> 1 -> 3 -> 2 -> 4 -> 2 -> 4 -> 2 ...
```

可以看到出现了环：

```txt
2 -> 4 -> 2
```

环的入口是 `2`，它也正是重复数字。

#### 为什么重复数字一定是环的入口？

因为重复数字意味着存在两个不同下标，它们指向同一个位置。

例如：

```txt
nums[3] = 2
nums[4] = 2
```

于是节点 `2` 同时被节点 `3` 和节点 `4` 指向：

```txt
3 -> 2
4 -> 2
```

在这个隐式链表中，这个拥有多个前驱节点的位置就是环的入口，因此环的入口值就是重复数字。

---

### 3.3. 第一阶段：让快慢指针在环中相遇

```js
let slow = nums[0],
  fast = nums[nums[0]]
```

这里相当于：

- `slow` 从下标 `0` 出发，走一步；
- `fast` 从下标 `0` 出发，走两步。

然后不断移动：

```js
while (slow !== fast) {
  slow = nums[slow]
  fast = nums[nums[fast]]
}
```

其中：

```js
slow = nums[slow]
```

表示慢指针每次走一步。

```js
fast = nums[nums[fast]]
```

表示快指针每次走两步。

由于链表中存在环，快指针最终一定会追上慢指针。注意，此时的相遇位置不一定是环的入口。

以示例 1 为例：

```txt
nums = [1, 3, 4, 2, 2]
```

初始化：

```txt
slow = nums[0] = 1
fast = nums[nums[0]] = nums[1] = 3
```

移动过程：

```txt
第一次：
slow = nums[1] = 3
fast = nums[nums[3]]
     = nums[2]
     = 4

第二次：
slow = nums[3] = 2
fast = nums[nums[4]]
     = nums[2]
     = 4

第三次：
slow = nums[2] = 4
fast = nums[nums[4]]
     = nums[2]
     = 4
```

快慢指针在节点 `4` 相遇。

---

### 3.4. 第二阶段：寻找环的入口

第一次相遇后，将慢指针重新放到起点 `0`：

```js
slow = 0
```

然后两个指针都改为每次走一步：

```js
while (slow !== fast) {
  slow = nums[slow]
  fast = nums[fast]
}
```

它们再次相遇的位置就是环的入口，也就是重复数字。

继续上面的例子，相遇时：

```txt
slow = 4
fast = 4
```

重置慢指针：

```txt
slow = 0
fast = 4
```

同步移动：

```txt
第一次：
slow = nums[0] = 1
fast = nums[4] = 2

第二次：
slow = nums[1] = 3
fast = nums[2] = 4

第三次：
slow = nums[3] = 2
fast = nums[4] = 2
```

两个指针在 `2` 相遇，因此返回：

```js
return 2
```

---

### 3.5. 为什么第二次相遇一定在环入口？

设：

- 从起点 `0` 到环入口的距离为 `μ`；
- 环的长度为 `λ`；
- 第一次相遇时，慢指针总共走了 `m` 步。

快指针走了 `2m` 步。因为它们在环中相遇，所以快指针比慢指针多走的 `m` 步一定是环长度的整数倍：

```txt
m = kλ
```

第一次相遇点相对于环入口的位置为：

```txt
m - μ
```

由于 `m` 是环长度的整数倍，因此：

```txt
m - μ ≡ -μ  (mod λ)
```

也就是说，从第一次相遇点再向前走 `μ` 步，恰好到达环入口。

第二阶段中：

- `slow` 从下标 `0` 出发，走 `μ` 步后到达环入口；
- `fast` 从第一次相遇点出发，走 `μ` 步后也到达环入口。

所以它们一定会在环入口相遇。

---

### 3.6. 完整代码解释

```js
var findDuplicate = function (nums) {
  // slow 走一步，fast 走两步
  let slow = nums[0]
  let fast = nums[nums[0]]

  // 第一阶段：寻找快慢指针的相遇点
  while (slow !== fast) {
    slow = nums[slow]
    fast = nums[nums[fast]]
  }

  // 第二阶段：寻找环的入口
  slow = 0

  while (slow !== fast) {
    slow = nums[slow]
    fast = nums[fast]
  }

  // 环的入口就是重复数字
  return slow
}
```

### 3.7. 复杂度分析

**时间复杂度：**

```txt
O(n)
```

快慢指针最多在线性数量的节点上移动。

**空间复杂度：**

```txt
O(1)
```

只使用了 `slow` 和 `fast` 两个变量。

同时，该算法只读取数组元素：

```js
nums[index]
```

不会修改原数组，满足题目的全部要求。

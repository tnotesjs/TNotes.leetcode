## 1. 📝 题目描述

- [leetcode](https://leetcode.cn/problems/minimum-operations-to-make-the-array-alternating/)

给你一个下标从 0 开始的数组 `nums`，该数组由 `n` 个正整数组成。

如果满足下述条件，则数组 `nums` 是一个 交替数组 ：

- `nums[i - 2] == nums[i]`，其中 `2 <= i <= n - 1`。
- `nums[i - 1] != nums[i]`，其中 `1 <= i <= n - 1`。

在一步 操作 中，你可以选择下标 `i` 并将 `nums[i]` 更改 为 任一 正整数。

返回使数组变成交替数组的 最少操作数。

---

示例 1：

```txt
输入：nums = [3,1,3,2,4,3]
输出：3
解释：
使数组变成交替数组的方法之一是将该数组转换为 [3,1,3,1,3,1]。
在这种情况下，操作数为 3。
可以证明，操作数少于 3 的情况下，无法使数组变成交替数组。
```

示例 2：

```txt
输入：nums = [1,2,2,2,2]
输出：2
解释：
使数组变成交替数组的方法之一是将该数组转换为 [1,2,1,2,1].
在这种情况下，操作数为 2。
注意，数组不能转换成 [2,2,2,2,2]。因为在这种情况下，nums[0] == nums[1]，不满足交替数组的条件。
```

---

提示：

- `1 <= nums.length <= 10^5`
- `1 <= nums[i] <= 10^5`

- 时间复杂度：$O(n)$，其中 $n$ 是数组长度
- 空间复杂度：$O(n)$，哈希表存储频率

算法思路：

- 分别统计偶数位和奇数位上各元素的出现频率
- 对每一组取出现次数最多的前两个元素
- 若偶数位和奇数位的最高频元素不同，直接使用；否则查看两种次高频组合取最小操作数
